Este trabajo e "Integration Process" son casi idénticos. Se diferencian en una única transformación que es la que se encarga de leer los ficheros XML. Esto es porque SourceForge y Assembla no tienen los mismos metadatos (tipos de ficheros) ni el vocabulario que emplean es común. Una vez leídos estos ficheros se almacena la información en la base de datos y se actualiza el estado de los proyectos.
